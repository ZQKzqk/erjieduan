相应状态吗
   100-199  表示连接继续

   200-299  表示各种意义上的成功

   300-399  表示重定向

   400-499  表示各种客户端错误

   500-599  表示各种服务端错


 了解一些常见的状态码
      + 100 表示连接继续

      + 200 表示通常意义上的成功
      + 201 表示创建成功
      + 204 表示删除成功
      + 目前基本上市场上的接口都是统一返回 200

      + 301 永久重定向
      + 302 临时重定向
      + 304 缓存

      + 403 权限不够
      + 404 找不到, 你写的地址不对

      + 500 常规的服务器错误
      + 501 过载或者服务器维护

当你的客户端请求 a 服务器的时候, 不是由 a 服务器给你返回的结果
      + 而是 a 服务器把你的请求改变成为了去请求 b 服务器, 有 b 服务器给         你返回结果


get  和  post  区别


    GET 和 POST 请求的区别(熟读并背诵全文)
      + GET
        => 语义偏向于获取
        => 携带参数的位置在地址后面直接拼接
        => 相对不安全, 明文发送
        => 发送的数据量比较小, 一般在 2KB 以下
        => 发送数据必须是 ASCII 码, 叫做 url 编码格式

      + POST
        => 语义偏向于提交
        => 携带参数的位置是在请求体中
        => 相对安全, 暗文发送
        => 发送的数据量比较大, 理论上来说无限大, 但是服务器可以限制
        => 理论上可以发送任何格式的字符串和文本内容, 但是要和请求头里面            的 content-type 配套
